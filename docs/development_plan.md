# C+ 开发路线图 (Development Roadmap)

C+ 致力于打造一个极简且高性能的 C 语言扩展，专注于稳定现有的 5 大核心语法。

## 已完成阶段 (Current Milestones)

- [x] **语义定义**: 支持 `let`, `unsafe`, `alias`, `bind`, `fork` 核心语法。
- [x] **所有权追踪**: 实现移动语义及生命周期失效检查。
- [x] **资源自动化**: 实现 RAII 析构逻辑自动生成。
- [x] **项目构建工具**: 实现了基于 `cplus.toml` 的项目自动化编译流程。
- [x] **稳定性与 C 兼容性增强 (Phase 6)**: 
    - [x] 优化 `alias mut` 到 C 指针的转换逻辑与自动解引用。
    - [x] 完善 `fork` 演化逻辑，实现扁平化结构体复刻。
    - [x] 实现了带源码片段和 `^` 标记的精确编译器报错系统。

---

## 正在进行与未来计划 (Future Roadmap)

### 阶段七：原生编译器后端 (LLVM)
- [ ] **LLVM IR 集成**: 摆脱中间 C 代码生成，直接通过 LLVM 生成机器码，实现更深度的硬件优化。
- [ ] **类型元数据增强**: 在位级别优化结构体布局，支持更高效的内存对齐。

### 阶段八：并发安全性调研
- [ ] 探索如何在 `let` 系统下提供更安全的跨线程资源传递（如 Send/Sync 语义初步引入）。
